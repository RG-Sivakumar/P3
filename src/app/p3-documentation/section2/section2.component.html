2.0 Application Settings Last updated:{{lastModifiedDate}}

Plannotate includes a number of application settings and options that are configurable per user.

  1. My Profile – Allows you to modify your personal profile settings.
  2. Security – Allows you to modify your Plannotate security settings on the iPad (not applicable for Plannotate web client).
  3. Sync – Allows you to application product online / offline synchronization settings on the iPad (not applicable for Plannotate web client).
  4. Support – Provides Plannotate support videos, documentation, and support ticket creation.
  5. About – Provides information on the current release of Plannotate.
  6. Sign Out – Enables you to log out of Plannotate.

2.1 My Profile Settings

The My Profile section of Project Management Dashboard Settings allows external users to change their personal information, including First Name, Last Name, Email Address and Password.

Note

As Plannotate leverages WJE’s Active Directory Server for internal employee information, WJE employees will not be able to make changes on this screen. For WJE employees, any desired changes will require you to follow the IT policies and procedures.

Non-WJE employees may modify their First Name, Last Name and / or password by selecting the edit icon to the right of any of those three options.

2.2 Security Settings

The Security section of Project Management Dashboard Settings on the iPad (not applicable for Plannotate web client) allows you to modify your Plannotate security settings. Security Settings differ from authentication / sign in to the application. Within the Security settings, users have the option to set an optional timeout period. If set, when the timeout period is reached, the application will require either Touch ID or Face Recognition to regain access to Plannotate.

Note

  • If your device has touch id capabilities, the toggle switch for this option will be enabled.
  • If your device has face recognition capabilities, the toggle switch for this option will be enabled.

There are two options for setting a timeout requiring you to unlock Plannotate.

  • Enable Touch ID – Toggle this option to allow Plannotate to be unlocked with Touch ID. This applies to iPads that are Touch ID capable (such as the one shown in the image above). If the device does not have this capability or if this capability is not enabled on the device, the toggle will be disabled and a message will indicate that the feature is not enabled or configured.
  • Enable Face Recognition – Toggle this option to allow Plannotate to be unlocked with Face Recognition. This applies to iPads that are Facial Recognition capable. If the device does not have this capability or if this capability is not enabled on the device (such as the one shown in the image above), the toggle will be disabled and a message will indicate that the feature is not enabled or configured.
  • Require After Duration – This setting allows a user to decide whether they want the application to lock after a specified period. The choices available include:
    • Every 30 Minutes
    • Every Hour
    • Every 12 Hours
    • Every 24 Hours
    • Never

Note

The default option is Never.

2.3 Sync Settings

The Sync section of Project Management Dashboard Settings on the iPad (not applicable for Plannotate web client) allows you to modify the application online / offline synchronization settings. Synchronization is one of the main features in Plannotate, and it allows users to sync their local copy of Plannotate with the Plannotate cloud. This process uploads all new data since the last sync to the Plannotate cloud, and downloads any new data to the iPad from the Plannotate cloud. Everything related to the project such as documents, toolbars, annotations, forms, form data, etc. will be synchronized when this process runs. This not only ensures that the information is safely stored on the Plannotate cloud, but also that it is available to any other Plannotate user on whatever device they are using from anywhere in the world.

  1. Manual Sync – Toggle this option on to allow Plannotate to sync only when manually triggered by selecting the Manual Sync option available throughout the application. When this option is toggled to on, the Wi-Fi Only and Wi-Fi + Cellular Network options will be toggled to off automatically.
  2. Wi-Fi Only – Toggle this option on to allow Plannotate to sync on a defined interval automatically whenever the device is connected to a Wi-Fi network. This will be transparent to the user. When this option is on, you may also perform a Manual Sync any time you desire. When this option is toggled to on, the Manual and Wi-Fi + Cellular Network options will be toggled to off automatically.
  3. Wi-Fi + Cellular Network – Toggle this option on to allow Plannotate to sync on a defined interval automatically whenever the device is connected to either a Wi-Fi or Cellular network. This will be transparent to the user. When this option is on, you may also perform a Manual Sync any time you desire. When this option is toggled to on, the Manual and Wi-Fi Only options will be toggled to off automatically.
  4. Sync Interval – This option is applicable when either the Wi-Fi Only or Wi-Fi + Cellular Network options is toggled to on.
    • Every 30 Minutes
    • Every Hour
    • Every 2 Hours
    • Every 8 Hours
    • Every 24 Hours

    Note

    The default option is Every 30 Minutes.

  5. Sync Photos and Videos on Wi-Fi Only – By default, photos and videos are synchronized along with all other content when a manual or scheduled sync occurs. When this setting is toggled to on, any newly added photos or videos will only be synchronized when on a Wi-Fi network. If the device is connected to a cellular network, photos and videos will remain on the device or on the Plannotate server but will not sync between until the device is connected to Wi-Fi and a sync occurs. This is useful in situations where you would like to sync data collected in the field but do not want the photo or video files to slow down the process.

2.4 Support

The Support section of Project Management Dashboard Settings provides access to Plannotate support videos, documentation and support ticket creation.

There are also several built in methods of sending information on the iPad (not applicable for Plannotate web client) to the support team.

  • Submit Log Files - Selecting this option will result in the log files from your local instance of Plannotate being sent to support for evaluation.
  • Submit Database Files - Selecting this option will result in the local Plannotate database being sent to support for evaluation.
  • Analyze / Troubleshoot - Selecting this option will result in an exact copy of your local Plannotate environment and data being sent to support for evaluation.

Warning

These three options should only be selected if they are requested by the Plannotate support team in response to an open support ticket.

It is important that you report any issues that you encounter so that we can reproduce and resolve as quickly as possible. We’d also like your feedback on enhancements or any other comments you may have throughout your use of the application.

In order to expedite the resolution of issues, we have integrated a support system so that you can send a support ticket right from the Plannotate App under Settings > Support > Submit A Support Request. You can also email support@wje.zendesk.com.

2.5 About Plannotate

The About section of Project Management Dashboard Settings provides information on the installed version of the Plannotate software. This is useful when support requests are submitted so that the application support team can determine if you are on the latest release of the software.

2.6 Sign Out of Plannotate

The Sign Out section of Project Dashboard allows you to log out of the application. This is useful if the device is shared among users, as actions performed within Plannotate are logged to the user that performs them. Therefore, it is beneficial for users to log on as themselves when using the application.

Danger

iPad users have the choice of keeping any local work that has not been synchronized with the Plannotate cloud or deleting the unsynchronized work. It may be desirable to remove the local work that has not been synchronized if you do not wish to save any of your changes. If this option is selected, the work will not be recoverable.
