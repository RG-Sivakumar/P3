6.0 Toolbar Builder Last updated:{{lastModifiedDate}}

The Toolbar Builder is the area where annotation icons are designed, named and collected into individual project toolbars. Toolbars are one of the most powerful features in Plannotate as they allow users to place any type of annotation on project documents and are fully customizable.

Users can use the Toolbar Builder to create new or edit existing toolbars. There are a variety of pre-defined annotation icons that can be used to create a toolbar. Toolbar annotations can be set to have a default form associated, and can also have prepopulated form values that are set by default when an annotation icon is placed onto a document.

Each new project comes with a Default toolbar, which can be modified or deleted.

  1. Project Title –References the name of the currently selected project.
  2. Toolbar Builder – The entry point to project toolbars.
  3. Support – Provides Plannotate support videos, documentation, and support ticket creation.
  4. Dashboard – Allows you to exit the project and return to the Project Management Dashboard.
  5. Toolbar List – This is the space where the list of all project toolbars will be displayed.
  6. Search – Enables you to search the displayed list of toolbars based on several search criteria. See Project Management Search for more information.
  7. Sort – Enables you to sort the displayed list of toolbars based on several sort options. See Project Management Sort for more information.
  8. Export – Allows users to export project content to a variety of formats. See Export for more information.
  9. Manual Sync – Initiates a synchronization of data with the Plannotate cloud, which uploads all new data since the last sync, and downloads any new data from the Plannotate cloud. See Project Management Manual Sync for more information. This icon is available only on Plannotate for the iPad, as the web client version is communicating real time with the Plannotate cloud. Plannotate on web client has an Apple icon in the top right corner to allow users to install Plannotate on their iPad.
  10. Add – Provides the ability to add new toolbars.

6.1 Toolbar List

The list of toolbars displays all project toolbars, sorted by default in alphabetical order.

When you first create a project, you will have a default toolbar, and as new toolbars are added, they will appear in a list view.

Once new toolbars have been added, or existing toolbars have been modified, they will either show in a blue or orange color. The list will default in alphabetical order, and can be searched and sorted to change the order displayed.

Note

  • Toolbars will show in an orange color if they have been created or modified on your device and have not yet been synchronized to the Plannotate cloud. Note that this is only relevant to Plannotate on the iPad, as the web client version is communicating real time with the Plannotate cloud. This allows users to work “offline” and sync to the Plannotate cloud at any time when they are back online, manually or on a schedule. See Sync Settings for more information.
  • Toolbars will show in a blue color if they have been fully synchronized to the Plannotate cloud.

6.1.1 Toolbar Options

There are two actions that can be performed on a toolbar.

  • Single tap – Selects and opens the toolbar into the Toolbar Builder.
  • Long hold – Opens a menu of toolbar options.

When a long hold is performed, a popup menu showing a variety of options will appear.

On the web client, a left click on your mouse button is used as a single tap, and a right click on your mouse button is used for a long hold.
  1. The title of the toolbar that was selected.
  2. Cancels out of this screen. You can also select anywhere off the menu to close the screen.
  3. Rename - Allows you to rename the toolbar.
  4. Manage Tags - Opens the Manage Tags screen.
  5. More Information - Displays toolbar metadata / details.
  6. Hide - Enables the toolbar to be hidden from use. See Hidden Content

6.1.1.1 Rename A Toolbar

The user that initially creates a toolbar will assign a name to the toolbar. In the event this name needs to change, this option can be selected.

When the Rename option is selected, the current name of the toolbar will appear, and by tapping on the data entry line, you will be able to modify or enter a new name.

  • Select Rename to change the name of the toolbar.
  • Select the X in the top right corner to cancel out of this screen, leaving the toolbar name unchanged.

6.1.1.2 Manage Tags

Tags are metadata assigned to a Plannotate object like a project, document, annotation, etc. By adding meaningful tags, users will be able to search for content more effectively.

When the Manage Tags option is selected, a popup will appear displaying any tags that have been assigned to the form. These will show up in orange, for example "Concrete".

  • Enter any number of tags on the data entry line, separated by a comma, and hit the enter key on your iPad keyboard to record them.
  • Select the X next to any tags in orange to remove them.
  • Select Save to save all the changes that were made.
  • Select the X in the top right corner to cancel out of this screen, leaving the tags unchanged.

6.1.1.3 Toolbar Information

The More Information screen shows metadata information associated with the toolbar and its contents.

  • The top section shows the toolbar name, created data, last modification date and the tags associated with the toolbar.
  • The main body of the screen shows how many toolbar annotations exist within the toolbar.
  • Select the X in the top right corner to cancel out of this screen.

6.2 Adding Project Toolbars

The Add Toolbar section of Toolbar Builder allows you to create new toolbars as well as make a copy of existing ones from within the project or from other projects which you have access. Any user with appropriate access to a project can create a new toolbar or copy other toolbar.

After tapping the + at the bottom right corner of the screen, a popup screen will appear allowing you to select from a few options.

  1. The title of the menu.
  2. Cancel out of this screen. You can also select anywhere off the menu to close the screen.
  3. New Toolbar - Add a new folder to the project.
  4. Duplicate Existing Toolbar - Add a new toolbar by making a duplicate of an existing toolbar within the same project.
  5. Import Toolbar From Other Projects - Add a new toolbar by making a duplicate of an existing toolbar from another project which you have access.

Note

If a toolbar is created with the same name as an existing toolbar, the name will be appended with a number, incremented by 1 each time this happens. For example, if a toolbar named "Crack" exists and is created for a 2nd time, it will receive the name "Crack (1)". If this were to be added again, it would be given a name "Crack (2)" and so on.

6.2.1 Adding a New Toolbar

After creating a project, you may want to add additional toolbars to capture annotation content. There are several methods to do this, the first of which is to add a brand new toolbar.

  • Provide a name for the new toolbar.
  • Select Add to add the toolbar.
  • Select the X in the top right corner to cancel out of this screen.

6.2.2 Duplicate an Existing Toolbar Within the Same Project

After creating a project, you may want to create additional toolbars to capture annotation content. There are several methods to do this, the second of which is to duplicate an existing toolbar within the same project.

Note

In order to select a toolbar to duplicate, check the box to the right of the desired toolbar. You may only select one toolbar at a time. Therefore, if you wish to create a copy of several toolbars, you will repeat these steps.

  • Select the toolbar you wish to duplicate from the list of toolbars within the project.
  • Select the Create Copy button.
  • Provide a name for the new toolbar.
  • Select Add to create the toolbar.
  • Select the X in the top right corner to cancel out of this screen.

6.2.3 Add an Existing Toolbar From a Different Project

After creating a project, you may want to create additional forms to capture annotation content. There are several methods to do this, the third of which is to add an existing toolbar from a different project project.

When you import a toolbar from another project, you will also import any forms that are being used by the toolbar as a default form for any of the annotation icons.

Note

In order to select a toolbar to add, check the box(es) to the right of the desired toolbar. You may select multiple toolbars at once.

Warning

In order to add toolbars from other projects, it is important that you first perform a sync to ensure that the projects and toolbars have been downloaded to your device.

  • Select the toolbars you wish to add from the list of toolbars within the projects which you have access.
  • Select the Import button.
  • Select the X in the top right corner to cancel out of this screen.

6.3 Toolbar Designer

Once a toolbar is selected, the toolbar designer screen appears, allowing you to create, modify and delete toolbar annotations.

The toolbar designer is divided into three primary areas: the available annotation icons, the body of the toolbar being created, and the properties of each toolbar annotation once selected.

  1. The name of the toolbar that was selected.
  2. The full list of annotation icons that are available to add to the toolbar.
  3. Support – Provides Plannotate support videos, documentation, and support ticket creation.
  4. Exit - Exists back to the main Toolbar Builder screen.
  5. The main body of the toolbar, where you will add, modify and organize your toolbar annotations.
  6. The properties area of the toolbar, where you will set up and modify each toolbar annotation property.
  7. Build – The build screen is the main screen of the Toolbar Designer.
  8. Preview – Preview allows you to see what the toolbar will look like as well as interact with the toolbar to test the functionality.
  9. Publish – Forms are not saved until the Publish button is selected.

6.4 Toolbar Annotation Icons

Every project comes with a Default Toolbar, which includes each of the 19 included annotation shapes. In addition, by default the simple shapes are blue with a blue border around them, and the drawn shapes will be drawn in blue outline with blue fill as applicable. Each shape can be customized through the annotation settings (see Settings).

The following table discusses each of the included annotation icons.

Note

  • The annotations that can be placed on a document as the exact shape of the annotation are simple shapes and include Circle, Octagon, Square, Triangle, Star, Diamond, Flag, Camera, Arrow and Text (a text annotation is a text bubble with a line extending from it).
  • The annotations that are added to documents as freehand drawings include Freehand-Line and Freehand-Area. The only difference between the two is that each Freehand-Area drawing is closed and and filled when the drawing is done.
  • In addition to freehand drawing, there are a number of annotations that allow for vertex drawing. Unlike freehand, vertex drawing involves setting points that are then connected by a line. These annotations include Polyline-Arrow, Polyline, and Polygon.
  • There are two drag-to-draw annotations that include Drawn-Rectangle and Drawn-Ellipse.

6.4.1 Adding Annotations to a Toolbar

Once a toolbar is created, annotations can be added to the toolbar by tapping on the annotation icons on the left. The available spaces are numbered starting with 1, and new annotations by default will be added to the next incremental number available.

Note

  • Annotation icons will be added in the order that they were selected. See Arranging Annotations to learn how to change the order.
  • The color, transparency and other various properties can be modified once the icon has been added to the toolbar. See Annotation Settings to learn how to modify the properties.
  • Annotation icons can also be added to toolbars through the document interface. This is covered in detail within the Documents section.

6.4.2 Arranging the Order of Annotations

Once annotations have been added to a toolbar, they can be moved to any desired order by dragging the desired annotation and dropping it in the new location.

6.4.3 Modifying Annotation Settings

Once annotations have been added to a toolbar, a variety of settings can be modified. These include:

  1. Close and Save - Select the X in the top right corner to save and close the setting window.
  2. Name - The name and label of the annotation. This text will appear on the toolbar.
  3. Stroke Color - Sets the stroke color for annotations that have an outside perimeter or line.
  4. Fill Color - Sets the fill color for annotations that have an inside area.
  5. Opacity - Sets the level of transparency for the fill color for annotations that have an inside area.
  6. Line Width - Sets the width of the annotation's line in pixels.
  7. Default Form - Establishes the default form that will be used when the annotation is added to a document and selected. This is an optional setting.
  8. Copy Annotation - Creates a duplicate copy of the annotation and places it on the toolbar. See Copy Annotation.
  9. Delete Annotation - Removes the annotation from the toolbar. See Delete Annotation.

Note

Certain annotations do not have all settings enabled, as they are not applicable. For example, the Line shape annotation does not have Fill properties. In the event that a setting is not applicable for an annotation icon, the setting will be disabled and appear in a lighter color to denote this.

6.4.4 Setting a Default Form

As annotations are added to document pages, additional information can be added to the annotation through the use of forms. Users can add data from any project forms that are available. In addition, users can pre-determine the default form that will be displayed when the annotation is selected within a document. As well, default values can be pre-populated so that they appear when the annotation is added.

Note

You may add as many default forms as you desire to each annotation icon. This is useful if you want to set pre-defined values for some of the form fields as a new annotation is added to a document. When adding more than one, the first one added will be the default.

6.4.5 Copy Annotations

Once an annotation is created, it can be easily duplicated into a new annotaion. The newly created annotation can then be modified as desired.

  • Select the desired annotation to duplicate.
  • Select the copy icon in the bottom right corner of the annotation settings window.
  • Provide a name for the newly created annotation.

6.4.6 Delete an Annotation

Once an annotation is created, it can be deleted at any time to be removed from the toolbar.

  • Select the desired annotation to duplicate.
  • Select the delete icon in the bottom right corner of the annotation settings window.
  • Confirm that the annotation should be deleted.

6.5 Preview a Toolbar

At any time during the creation or modification of a toolbar, users can preview what the toolbar will look like when Published. This will allow the user to confirm that the toolbar is working as expected.

Note

In order to confirm that the default form values work as expected, you will need to add the annotation icon to a document. The Toolbar builder does not have a capability for doing this.

6.6 Publish a Toolbar

Once the toolbar is complete and ready for use, users can Publish the toolbar. If the toolbar is new, users will now be able to use it. If the toolbar contains modifications to an existing toolbar, the modifications will now be seen.

Danger

If a toolbar is exited before publishing, all changes will be lost. This may be desired if the user is testing or trying new things. In the event a user selects Exit and has not published the toolbar, a warning will be presented confirming this action is desired.
