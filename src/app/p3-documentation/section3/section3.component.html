3.0 Project Management Last updated:{{lastModifiedDate}}

The Project Management Dashboard is the entry point into a Plannotate project. Users will see a list of the projects which they have access based upon their login credentials.

  1. Title –References where you are within the Project Management Dashboard, such as My Projects, Recent Projects.
  2. My Projects – Provides a list of all Plannotate Projects which you have access.
  3. Recent Projects – Provides a list of the last several Plannotate Projects which you have accessed.
  4. Favorite Projects – Provides a list of all Plannotate Projects which you have marked as a favorite.
  5. Hidden Projects – Provides a list of all Plannotate Projects which you have marked as hidden.
  6. Sidebar Expand / Collapse – The Plannotate sidebar on the Project Dashboard screen can be expanded and collapsed to maximize real estate.
  7. Projects – This is the space where the list of projects will be displayed based upon which tab you select from the left side toolbar.
  8. Search – Enables you to search the displayed list of projects based on several search criteria.
  9. Sort – Enables you to sort the displayed list of projects based on several sort options.
  10. Manual Sync – Initiates a synchronization of data with the Plannotate cloud, which uploads all new data since the last sync, and downloads any new data from the Plannotate cloud. This icon is available only on Plannotate for the iPad, as the web client version is communicating real time with the Plannotate cloud. Plannotate on web client will have an Apple icon in the top right corner as discussed in the installation section.
  11. Add – Provides the ability to add new projects.

3.1 My Projects

The My Projects area of the Project Management Dashboard provides a list of all projects which you have access. Any new projects you create will automatically appear in this list, and as you are added to other projects, they will also display in the list.

When you first log into Plannotate, you may not have any projects assigned to you or created. Once you are added to your first project, it will appear in a list view.

Note

Upon signing into Plannotate for the first time, it is advisable to click the manual sync button in order to download any projects that you have been provided access to. These will not appear in the list until you perform a synchronization.

Once projects have been added, they will either show in a blue or orange color. The list will default in alphabetical order, and can be searched and sorted to change the order displayed.

Note

  • Projects will show in an orange color if they have been created or modified on your device and have not yet been synchronized to the Plannotate cloud. Note that this is only relevant to Plannotate on the iPad, as the web client version is communicating real time with the Plannotate cloud. This allows users to work “offline” and sync to the Plannotate cloud at any time when they are back online, manually or on a schedule. See Sync Settings for more information.
  • Projects will show in a blue color if they have been fully synchronized to the Plannotate cloud.

3.2 Project Options

There are two actions that can be performed on a project.

  • Single tap – Selects and opens the project.
  • Long hold – Opens a menu of project options.

When a long hold is performed, a popup menu showing a variety of options will appear.

On the web client, a left click on your mouse button is used as a single tap, and a right click on your mouse button is used for a long hold.
  1. The title of the project that was selected.
  2. Cancels out of this screen. You can also select anywhere off the menu to close the screen.
  3. Add to Favorite - Adds the selected project to the list of favorites. If the project is already a favorite, this option will read “Remove from Favorite”. See Favorite Projects.
  4. Rename Project - Allows you to rename the project.
  5. Manage Tags - Opens the Manage Tags screen.
  6. Project Information - Displays project metadata / details.
  7. Hide - Enables the project to be hidden from the My Projects, Recent Projects, and Hidden Projects screens. See Hidden Projects
  8. Manage Users - Opens the Manage Users area in order to establish access and security rights to the project.

3.2.1 Add to Favorites

The Add to Favorite option designates the selected project as a personal favorite. This means that the project will show up in the Favorite Projects section of the Project Management Dashboard, and will be denoted by a special icon.

If the project options are selected on a project designated as a favorite, this option will change to read "Remove from Favorite". If the Remove from Favorites option is selected, the project will remain in the My Projects and Recent Projects as applicable but will no longer appear in the Favorite Projects. The icon will also change back to the default project icon.

3.2.2 Rename a Project

The user that initially creates a project will assign a name to the project. In the event this name needs to change, this option can be selected.

When the Rename Project option is selected, the current name of the project will appear, and by tapping on the data entry line, you will be able to modify or enter a new name.

  • Select Rename to change the name of the project.
  • Select the X in the top right corner to cancel out of this screen, leaving the project name unchanged.

3.2.3 Manage Tags

Tags are metadata assigned to a Plannotate object like a project, document, annotation, etc. By adding meaningful tags, users will be able to search for content more effectively.

When the Manage Tags option is selected, a popup will appear displaying any tags that have been assigned to the project, document, etc. These will show up in orange, for example "Concrete".

  • Enter any number of tags on the data entry line, separated by a comma, and hit the enter key on your iPad keyboard to record them.
  • Select the X next to any tags in orange to remove them.
  • Select Save to save all the changes that were made.
  • Select the X in the top right corner to cancel out of this screen, leaving the tags unchanged.

3.2.4 Project Information

The project information screen shows metadata information associated with the project.

  • The top section shows the project name, created data, last modification date and the tags associated with the project.
  • The main body of the screen shows how many occurrences of the various types of objects exist within the project.
  • Select the X in the top right corner to cancel out of this screen.

3.2.5 Hide

The Hide option designates the selected project to be hidden from My Projects, Recent Projects and Favorite Projects. This is a personal setting, and the selected project will only be hidden for the user that selects this option. Other users of the project can choose to hide or show and are not impacted by the decision to hide by another user. By selecting Hide, the project will show up in the Hidden Projects section of the Project Management Dashboard, and will be denoted by a special icon.

If the project options are selected on a project designated as hidden, this option will change to read "Unhide". If the Unhide option is selected, a project will move back to the My Projects and any of the other applicable views within the Project Management Dashboard. The icon will also change back to the default project icon.

3.2.6 Manage Users

Once a project has been created, anyone with the proper security can add, modify, and remove user access to the project.

Note

The project must be synchronized at least once before users can be added so that the Plannotate Cloud is aware that the project exists. If a new project is created and not yet sync'd, your search for users will not generate any results.

  1. The title of the project that was selected.
  2. The Add Users search box allows you to search for any existing Plannotate user, including any active WJE employee and any external users that have been granted access to Plannotate. Type any portion of the desired email address or name and tap the search icon.
  3. The search results will appear in this section, allowing you to select users to add to the project.
  4. External users can be added to a project by entering their first name, last name and email and then selecting the Add New User.
  5. All project members will appear in this section, and you can select on the various security columns to modify each individual’s security.
    • View – allows the user to view the project and contents but not make changes.
    • Edit – allows the user to make additions, edits and deletions to the project. They will not be allowed to add, remove or modify project security or users.
    • Admin – grants full rights to the project, including security features.
  6. Cancels out of this screen. You can also select anywhere off the window to close.

3.3 Recent Projects

The Recent Projects area of the Project Management Dashboard provides a list of the last several projects which you have accessed.

When you first log into Plannotate, you will not have any projects in the Recent Projects area. Once you open a project, it will appear in a list view.

Once Projects have been opened, they will either show in a blue or orange color. The list will display the last 4 projects that were opened and default in descending modified date order, and can be searched and sorted to change the order displayed.

  • Projects will show in an orange color if they have been created or modified on your device and have not yet been synchronized to the Plannotate cloud. Note that this is only relevant to Plannotate on the iPad, as the web client version is communicating real time with the Plannotate cloud. This allows users to work “offline” and sync to the Plannotate cloud at any time when they are back online, manually or on a schedule. See Sync Settings for more information.
  • Projects will show in a blue color if they have been fully synchronized to the Plannotate cloud.

3.4 Favorite Projects

The Favorite Projects area of the Project Management Dashboard provides a list of projects which you have designated as favorites. Favorites are unique to each user, so the favorites of any one user are not automatically designated as a favorite of any other user.

When you first log into Plannotate, you will not have any projects in the Favorite Projects area. Once you designate a project as a favorite project, it will appear in a list view.

Once Projects have been designated as favorites, they will be denoted with a special icon and will either show in a blue or orange color. The list defaults in descending modified date order, and can be searched and sorted to change the order displayed.

  • Projects will show in an orange color if they have been created or modified on your device and have not yet been synchronized to the Plannotate cloud. Note that this is only relevant to Plannotate on the iPad, as the web client version is communicating real time with the Plannotate cloud. This allows users to work “offline” and sync to the Plannotate cloud at any time when they are back online, manually or on a schedule. See Sync Settings for more information.
  • Projects will show in a blue color if they have been fully synchronized to the Plannotate cloud.

Note

Projects are designated as Favorites on a per user basis, and therefore any projects marked as Favorite by one user will not automatically be marked as a favorite by any other user.

3.5 Hidden Projects

The Hidden Projects area of the Project Management Dashboard provides an area for users to place projects that they do not want included in the My Projects, Recent Projects or Favorite Projects area. They are not deleted from the Plannotate cloud. Hidden projects are unique to each user, so the ones hidden by any one user are not automatically hidden by any other user.

When you first log into Plannotate, you will not have any projects in the Hidden Projects area. Once you designate a project as a hidden project, it will appear in a list view.

Once Projects have been designated as hidden, they will be denoted with a special icon and will either show in a blue or orange color. The list defaults in descending modified date order, and can be searched and sorted to change the order displayed.

  • Projects will show in an orange color if they have been created or modified on your device and have not yet been synchronized to the Plannotate cloud. Note that this is only relevant to Plannotate on the iPad, as the web client version is communicating real time with the Plannotate cloud. This allows users to work “offline” and sync to the Plannotate cloud at any time when they are back online, manually or on a schedule. See Sync Settings for more information.
  • Projects will show in a blue color if they have been fully syncrhonized to the Plannotate cloud.

Note

Projects are hidden on a per user basis, and therefore any projects hidden by one user will not automatically be hidden by any other user.

3.6 Project Management Dashboard Sidebar

The sidebar on the Project Management Dashboard screen can be expanded and collapsed to maximize real estate. This is useful when looking at projects with long names.

By default. the sidebar will be expanded, showing both an icon and text description. Selecting the orange arrow icon will allow users to shink and expand the sidebar.

3.7 Creating New Projects

The Add New Project section of Project Management Dashboard allows users to create a new project. Any user with access to Plannotate can create a new project, add content, and add other users. By default, the user who creates the project will have administrator rights to the project.

After tapping the + at the bottom right corner of the screen, a popup screen will appear allowing you to give the new project a name.

  • Select Add to create the project.
  • Select the X in the top right corner to cancel out of this screen. You can also select anywhere off the popup screen to close.

Note

If a project is created with the same name as an existing project, the name will be appended with a number, incremented by 1 each time this happens. For example, if a project named "2020.0000.1 – Project ABC" exists and is created for a 2nd time, it will receive the name "2020.0000.1 – Project ABC (1)". If this were to be added again, it would be given a name "2020.0000.1 – Project ABC (2)" and so on.

3.8 Search

The project search feature works the same within the My Projects, Recent Projects, Favorite Projects, and Hidden Projects sections.

  • Tap on the Search icon in the top bar.
  • Type the text to search for, which can include any part of the:
    • Text within the name of the project.
    • Text within the name of any of the project tags.
    • Create date.
    • Modified date.

Note

As you begin entering text into the search box, the list of projects will begin to filter based on the information you are entering.

When searching by create date or modified date, the syntax to search is "mm/dd/yyyy".

3.9 Sort

The project sort feature works the same within the My Projects, Recent Projects, Favorite Projects and Hidden Projects sections.

  • Tap on the Sort icon in the top bar.
  • Select the desired sort option. This can include:
    • Text based on the name of the project, sorted A to Z
    • Text based on the name of the project, sorted Z to A
    • Date based on the project creation date and time, sorted oldest to newest
    • Date based on the project creation date and time, sorted newest to oldest
    • Date based on the project last modification date and time, sorted oldest to newest
    • Date based on the project last modification date and time, sorted newest to oldest
  • The list of projects will now be re-sorted, based upon your selection. In this example, Name (Z to A) was selected.

Note

Sorting the list of projects will retain the display until the screen is changed, at which time it will return to the default sorting option, which is by name sorted A to Z.

3.10 Manual Synchronization

Plannotate on the iPad can be used online or when offline and disconnected from a network. This is beneficial in instances where network connectivity is not available, slow or unstable. Anytime a user is connected to a network, whether on cellular or Wi-Fi, a manual synchronization can be started. This will push any modifications on the iPad to the Plannotate cloud, and retrieve any modifications that were not previously downloaded to the iPad.

Plannotate on web client communicates in real time with the Plannotate cloud and therefore does not have a manual synchronization feature.

Note

Users will always know what projects have and have not been synchronized with the Plannotate cloud based on the text color of the project name.

Any work that is completed locally and not synchronized to the Plannotate Cloud will remain on the iPad until the user signs out of Plannotate. It is highly recommended that users periodically synchronize throughout the duration of a project to avoid any data loss. Note that this is only relevant to Plannotate on the iPad, as the web client version is communicating real time with the Plannotate cloud.

  • Blue text indicates that a project has been synchronized with the Plannotate cloud.
  • Orange text indicates a project has not been synchronized with the Plannotate cloud.
  • When the manual sync button is selected, the iPad will connect to the Plannotate cloud, determine any unsynchronized changes, and conduct a merge to ensure both the iPad and Plannotate cloud are in sync
