8.0 Annotations Last updated:{{lastModifiedDate}}

Annotations allow users to add markups to a document. Subsequently, users can link document pages to annotations or record data such as form entries and photos. Users can also add new annotation icons that have been created within a document to an existing or new toolbar.

  1. Toolbar Title Button – Displays the title of the selected toolbar. Tap to switch to a different toolbar.
  2. Toolbar – List of annotations for the selected toolbar.
  3. Move Toolbar - Hold on the double line and then drag and drop the toolbar onto the top, bottom or right side of the screen.
  4. Data Panel Size Adjuster – Adjusts the size of the data panel by dragging.
  5. Forms Tab – Data Panel tab that displays forms and form data related to the selected annotation.
  6. Media Tab – Data Panel tab that displays photos and stubs related to the selected annotation.
  7. Links Tab – Data Panel tab that displays linked documents and URLs related to the selected annotation.
  8. Tags Tab - Data Panel tab that displays the tags assigned to an annotation and allows for the adding and removing of tags.
  9. Properties Tab - Data Panel tab that displays and allows users to modify the annotation properties.
  10. Page Navigation - Navigate through the various pages within a document.

8.1 Toolbar Title Button

The toolbar title button displays the title of the current / selected toolbar. Projects can have as many toolbars as required, and the display on the button will indicate which one is currently being used.

As users begin annotating documents, they can switch toolbars at any time by tapping on the button. This will bring up a dynamic list of other toolbars in the project, along with several action buttons.

  1. Select any of the rows in the list in order to switch toolbars.
  2. Click the Edit link in order to open the Toolbar Builder in order to make edits.

Note

The checkmark indicates the currently selected Toolbar.

8.2 Toolbar

Toolbars are discussed in detail in Chapter 6 - Toolbar Builder. Projects can create as many toolbars as required.

When you first open a document, the last used Toolbar will appear. Each toolbar contains 0 or more annotation icons, which are thumbnail representations of what the annotation will look like when placed upon a page.

Toolbars are discussed in detail throughout the application documentation, including how to modify the toolbar annotations, hiding and showing the annotation names, etc.

8.3 Move Toolbar

Toolbars can be placed in 3 areas within the documents user interface depending on each user's personal preference.

In order to move the toolbar, hold on the triple line at the end of the toolbar and then drag and drop the toolbar. As you start to move the toolbar, three areas will appear in gray, indicating locations where you can drop the toolbar.

  • On the top of the current document.
  • To the right of the current document.
  • On the bottom of the current document.

8.4 Resize Data Panel

The data panel anchored to the left of the document user interface can be resized in order to minimize or maximize real estate on the document.

In order to resize the data panel, hold on the double line in the top right corner of the data panel, and then drag and drop the data panel. As you start to move the data panel, the document will adjust accordingly.

8.5 Placing Annotations on Documents

Annotations can be placed onto a document by tapping on the desired annotation and then tapping on the location on the document where you wish to place the annotation. Depending on the type of annotation, additional steps may be required.

The following sections discussion how to place each type of annotation onto a document page.

  • Simple Shapes
  • Text
  • Freehand
  • Polyline / Polygon
  • Drawn Shapes
  • Lines

8.5.1 Placing Simple Annotations

Simple Annotations can be placed onto any document page by simply tapping on the shape on the toolbar and then tapping at the desired location on the page.

  • Tap on the annotation shape on the toolbar.
  • Tap on the location on the document page where the annotation shape is to be placed.

8.5.2 Placing Text Annotations

Text Annotations can be placed onto any document page by simply tapping on the shape on the toolbar and then tapping at the desired location on the page. Once placed onto the document, a window opens to allow users to type whatever text they desire.

  • Tap on the annotation shape on the toolbar.
  • Tap on the location on the document page where the annotation shape is to be placed.
  • Enter the desired text to appear inside the text annotation.

8.5.3 Placing Freehand Annotations

Freehand Annotations allow users to draw freely on a document, saving the drawing(s) as a single annotation or multiple annotations.

  • Tap on the annotation shape on the toolbar.
  • Tap on the location on the document page where the annotation shape will be started.
  • Draw the desired shape(s).
  • Determine if the drawing(s) should be saved as one shape or many shapes.

Note

At any time you can select Undo in order to undo the last action(s) or the Cancel button to discard your drawings.

8.5.4 Placing Polyline / Polygon Annotations

Polyline and Polygon Annotations allow users to place simple line segments onto a document.

  • Tap on the annotation shape on the toolbar.
  • Tap on the location on the document page where the annotation shape will be started.
  • Continue to tap on the document page to place points that will form a line segment.
  • Select Done to save the annotation.

Note

At any time you can select Undo in order to undo the last action(s) or the Cancel button to discard your drawings.

8.5.5 Placing Drawn Shape Annotations

Drawn Shape Annotations allow users to place a pre-defined shape drawn onto a document using a dragging motion to determine the size.

  • Tap on the annotation shape on the toolbar.
  • Tap on the location on the document page where the annotation shape will be started.
  • Drag until you have achieved the desired shape and size.

8.5.6 Placing Line Annotations

Line Annotations allow users to place simple line segments onto a document.

  • Tap on the annotation shape on the toolbar.
  • Tap on the location on the document page where the annotation shape will be started.
  • Drag until you have achieved the desired shape and size.

Note

You may continue drawing lines until you are satisfied with the annotation drawing, and then select OK to save. You can also the Cancel button to discard your drawing.

8.6 Annotation Options

Once an annotation has been placed on a page, there are a number of options users have to take action on the shape itself.

  1. Delete – Delete the selected annotation(s).
  2. Copy With Data – Copy the selected annotation(s) with Form data, Link data, Tag data and Properties preserved.
  3. Copy Without Data - Copy the selected annotation(s) without Form data, Link data or Tag data.
  4. Move to Layer – Move the selected annotation(s) to another Layer within the document.
  5. Use as Toolbar – Create a new toolbar using the selected annotation(s) .
  6. Add to Toolbar With Data – Add the selected annotation(s) with Form data, Link data, Tag data and Properties preserved to an existing Toolbar.
  7. Add to Toolbar Without Data – Add the selected annotation(s) without Form data, Link data or Tag data to an existing Toolbar.
  8. Add to Toolbar Without Data – Add the selected annotation(s) without Form data, Link data or Tag data to an existing Toolbar.
  9. Resize and Rotate - Resize, repoint and / or rotate the selected annotation(s).
  10. Set as Base Icon Size - Establish a size that all new simple shape annotation icons will be set to when added to the document.

Note

As multiple annotations can be selected at one time, many of the above actions can be performed on several annotations at once.

8.6.1 Delete Annotations

Once an annotation has been placed on a page, it can be deleted at any time if it is no longer desired.

  • Select the annotation(s) to delete.
  • Long tap to bring up the additional options menu.
  • Select Delete Annotation.
  • Select Confirm at the confirmation screen that appears.

Note

As multiple annotations can be selected at one time, the above action can be performed on several annotations at once.

8.6.2 Copy With Data

Each annotation can be associated with one or more forms, images, links, tags and properties. It is often desired to make a copy of an annotation in order to place it somewhere else on the page or on another page within the same document. Copy With Data is one of the two ways to achieve this.

Copy With Data makes a copy of the selected annotation(s) including the associated forms, links, tags and properties. This copy can then be placed onto the document pages as many times as required.

Note

Images are unique to the specific location of an annotation and therefore are not replicated when a Copy With Data action is performed.

  • Select the annotation(s) to copy.
  • Long tap to bring up the additional options menu.
  • Select Copy With Data.
  • Select the annotation copy from the toolbar, which has been placed on top of whatever the current toolbar is.
  • Tap to place a copy of the annotation onto the document.
  • Repeat as desired, switching pages if necessary.
On the web client, a left click on your mouse button is used as a single tap, and a right click on your mouse button is used for a long hold.

Note

As multiple annotations can be selected at one time, the above action can be performed on several annotations at once.

8.6.3 Copy Without Data

8.7 Forms Tab

The forms tab resides on the data panel and is used to populate information on an annotation or multiple annotations at a time. This area only becomes active when one or more annotations are selected.

Once an annotation is selected, the default form for the document will display unless the annotation has been assigned a default icon. Learn more about setting a default form in the Default Form section of Toolbar Builder.

If multiple annotations are selected, this will be indicated by an additional notation at the top of the data panel, indicating how many annotations are selected.

Warning

When multiple annotations are selected, any information entered into a form or forms will be applied to all selected annotations. If any information previously exists for a form field in one of the selected annotations, it will be overwritten.

If a project has more than one form, users can switch forms by tapping on the button labeled with the name of the current form.

  • Select from the list of forms that appear on the popup to switch forms.
  • Select Edit to enter Form Designer, where you can modify the form if desired.
  • Select the i icon to detach the form from the annotation icon.

    • Warning

    If you detach a form from an annotation, all form values will be reset if you decide to use the detached form again with the annotation.

  • The checkmark indicates which form is set as the default / current form for the annotation.

Note

A badge icon will appear on top of the form tab icon at the bottom of the data panel when one or more forms have been populated for the selected annotation(s).

If multiple annotations are selected, this will be indicated by an additional notation at the top of the data panel, indicating how many annotations are selected.

8.8.4 Add Stub Photo

Often times it is desirable to use a separate camera to take annotation photos. When this is preferable, users require a way to add the photos to their project and associate them with the desired annotation. Rather than having to go back to the project, locate the annotation, and then add the photo(s) after the work is complete and the images have been taken off of the camera, users can create stub photos. Stub photos are placeholders that enable you to save the exact name of the photo that will be added to the project at a later date.

Stub Photos are reconciled by matching the exact name (case sensitive) of an image later uploaded to a project to the name that was given to a stub photo when added to an annotation.

  1. Use Name / Date as Prefix - If this option is selected, the name of the photo prefix will change to the user's email address used to log in (prior to the @domain), an "_", the current date in the format yyyymmyy, an "_", the photo number, and ".jpg". For example, "jdoe_20210115_0001.jpg".
  2. Set Prefix - As an alternative to using Name / Date a a prefix, users can specify their own desired prefix. Often time digital cameras use a prefix of "IMG_".
  3. Set Number - This will be the first photo number used when creating the stub file, and will be incremented by 1 for each additional stub photo desired.

    4. Note

    Plannotate retains the last photo number used when creating stub files so as additional stub photos are created, they will not conflict in numbering with ones that were previously added.

  4. Number of Stubs - By default, 5 stub photo names will be generated. This number can be increased or decreased as desired.

    5. Note

    Regardless of the number of stubs created and displayed to the right, no stub photos will be saved unless they are selected by tapping on the checkbox to the right of the names.

  5. Number - Users can increase or decrease the photo number by selecting the - or + buttons. This is an alternative to typing in a new number in the Set Number field.
  6. Stubs - Users can increase number of stub names created by selecting the +1 through +5 buttons. This is an alternative to typing in a new number in the Number of Stubs field.
  7. Stub file names are created and displayed based upon the actions taken in the steps above. In order to add the Stub Photos to an annotation, you must select one or more of the names that appear in the list.
  8. After selecting one or more stub file names, select the Add button to add the stub photo placeholders to the annotation.
  9. Select the X in the top right corner to exit without creating any stub photos.

If multiple annotations are selected, the photo(s) will be added to all annotations.

8.8.4.1 Reconciling Stub Photo

The process to reconcile stub photos involves matching the exact name of the stub file(s) with an image file of the same name. The images must be of type jpg (Joint Photographic Group), with an extension of .jpg in lowercase.

Warning

Reconciling stubs requires the file name and the stub name to match exactly, and is case sensitive. A stub named IMG_0001.JPG will not be reconciled with a file named IMG_0001.jpg, as the file extensions do not match.

Note

When Plannotate is used on an iPad, all work is preformed offline until a synch with the Plannotate cloud takes place, either manually or on an automated schedule. If a stub photo is added to a project and the project is not subsequently synchronized, the Plannotate cloud will not know it exists and therefore will be unable to reconcile.

  • Once a stub file or files are added to a project, synchronize the project so that the Plannotate Cloud is aware that they exist.
  • Navigate to Plannnotate's web client interface, log in and access the applicable project.
  • Select the Reconcile Stub Photos icon in the top right corner.
  • Drag and drop the desired photo(s) to be reconciled, or select the cloud icon to enable selecting the file(s) from your computer.
  • Select the Upload File button when you have added all desired photos to be reconciled.
  • Perform a sync on the Plannotate iPad application in order to get the latest information from the Plannotate cloud, including the reconciled photos.

Warning

If any of the photos selected have a file extension other than .jpg, an error message will appear and the file will not be uploaded. If multiple files are selected, those that match the .jpg file extension will be uploaded while those that do not will not.

8.9.1 Link to Document Page

  • Determine if the annotation should link to:
    • the selected document page.
    • an exact copy of the entire document. If this option is selected, the target document will be duplicated with all markups, and the link will reference the new copy.
    • a new copy of the entire document. If this option is selected, the target document will be duplicated without any markups, and the link will reference the new copy.

8.11 Properties Tab

The properties tab resides on the data panel and is used to populate information on an annotation or multiple annotations at a time. This area only becomes active when one or more annotations are selected.

  1. Entering text in this box will change the name of the selected annotation.
  2. Entering text in this box will add a label to the selected annotation. This label text will appear on top of the annotation and will be visible on the document.
  3. Tapping on the box to the right of Shape allows users to change the shape of the selected annotation. Users can pick from any of the available shapes that appear in the popup.
  4. The stroke color is the color of the line around the shape, or comprising the shape in the case of lines that only have a perimiter. It is also the color that will be used for the Label text.
  5. The fill color is the color of the interior of the shape and is not applicable to shapes that only have a perimeter.
  6. The opacity of a shape determines how transparent the fill color will appear, and is a range of 0 to 1, where 0 is completely transparent and 1 is completely solid.
  7. The line width of a shape determines how thick the line around the shape appears, or how thick the line that comprises the shape appears.

If multiple annotations are selected, this will be indicated by an additional notation at the top of the data panel, indicating how many annotations are selected.

Warning

When multiple annotations are selected, any information entered into a form or forms will be applied to all selected annotations. If any information previously exists for a form field in one of the selected annotations, it will be overwritten.

8.12 Page Navigation

When a document has more than one page, there are several different methods for navigating between them.

When you first open a document, the PDF will appear without any markups applied. Users can then beginning adding annotations to the document through the various Toolbars.

  • Users can change pages using the left and right arrows on either side of the page number at the top of the document screen. This will switch one page at a time with each tap on the < or >.
  • Tapping on the "X/Y", signifying the current page (X) out of total pages (Y) in the document, pops up an additional screen that will allow users to select the page to navigate to, or to search for the desired page.
  • Selecting the More Options will provide the option to Switch Pages, which achieves the same result as tapping on the "X/Y".
