7.0 Documents Last updated:{{lastModifiedDate}}

Project documents are where the core of the application lies. This is where users can interact with documents and add annotations to a page. Subsequently, users can link document pages to annotations or record data such as form entries and photos. Users can also add new annotation icons that have been created within a document to an existing or new toolbar.

  1. Document Space – Displays the pdf of the currently selected page to be worked on.
  2. Data Panel – Displays the data that is tied to each annotation, excluding the Layers tab.
  3. Back Button - Return to the Project Screen.
  4. Document Title – Displays the document name and the current page out of the total number of document pages.
  5. Search – Enables you to search for annotations based on several search criteria.
  6. Move - Allows annotations to be moved through drag and drop when selected.
  7. Undo - Allows users to remove the last action(s) performed, one by one. NOTE: This feature is not currently available on web client and is planned for a future release.
  8. Redo - Works as an undo for the undo action. NOTE: This feature is not currently available on web client and is planned for a future release.
  9. Multi-Select - When selected, users can pick multiple annotations on the page in order to perform a common set of actions.
  10. Layers – Displays the layer panel popup.
  11. Export - Enables the document to be exported to PDF in order to save or email.
  12. Manual Sync – On Plannotate for iPad, this initiates a synchronization of data with the Plannotate cloud, which uploads all new data since the last sync, and downloads any new data from the Plannotate cloud. See Project Management Manual Sync for more information. On Plannotate for Web, this will refresh the document and pull any changes that have taken place in the cloud since the document was opened, such as those resulting from another user's work.
  13. More Options – Displays a popup with multiple extra options for the document.

7.1 Document Space

The document space is the area where PDF documents that have been added to the project will appear. This area allows users to zoom in and out on the document, as well as mark up the document with an unlimited number of annotations.

When you first open a document, the PDF will appear without any markups applied. Users can then beginning adding annotations to the document through the various Toolbars.

7.2 Data Panel

The data panel area stays permanently on the left side of the Document screen and allows users to modify annotations. This includes adding form content, images, links, tags and property modifications. Each of these topics are covered in a separate section of this document.

Note

    The data panel will remain empty until one or more annotations are selected.

7.3 Search

Users can search within a document in order to locate annotations that meet the search conditions selected. This helps to identify annotations more easily, especially when a large number of annotations existing within the pages of a document.

When a user selects the search button, a separate window will appear on top of the Data Panel with various options. As annotations are located and selected by marking the checkbox to the right of the annotation name, they will remain selected once the Search window is closed. This enables users to modify the selected annotation(s) usin the Data Panel without having to reselect them.

  1. Tap on the Search icon to open and close the Search window.
  2. Text field for adding search terms.
  3. Filters that can be applied to the search.
    • All - The default filter, which allows the search to be conducted with all of the other filters allowed.
    • With Photos - This will identify any annotations that have photos added to the Media Tab.
    • Form Values - This will identify any annotations where a form has been attached.
    • Tag Values - This will identify any annotations where one or more tags have been assigned.
    • With Stubs - This will identify any annotations that have stub photos added to the Media Tab.
  4. The reults of the search will appear in the list below. Checking any of the annotations will result in them being selected on the document.

Note

    The text field currently applies only when the All option is selected in the Filters. This will expand in future releases.

Warning

The filter options work as a union, meaning that if multiple boxes are checked, the annotations must meet all of the filter requirements. For example, an annotation with photos and stub photos will be found if the With Photos box or the With Stubs box is selected. It will also be found if both boxes are selected. Another annotation with only photos and no stub photos will not be found if both boxes are selected.

7.4 Moving Annotations

Once an annotation is placed onto a document page, it can be relocated to another place within the same document page. This is done by selecting the move button in the top toolbar. The button must be selected in order to move an annotation in order to prevent accidentally moving the annotation when performing some other action.

Steps for iPad

  • Select the move icon in the top toolbar.
  • Tap on the annotation icon to be moved in order to select it.
  • Drag the annotation icon with your finger to the desired location.
  • Select the move icon in the top toolbar when finished.

Steps for Web Client

  • Select the move icon in the top toolbar.
  • Double click on the annotation icon to be moved in order to select it.
  • Drag the annotation icon with your mouse to the desired location.
  • Select the Done icon on the right side of the screen when finished.

You may also wish to move several annotations at the same time. This is possible using the combination of the move icon Mutli-Select icon.

Steps for iPad

  • Select the multi-select icon in the top toolbar.
  • Tap on each annotation icon to select them, or drag to select multiple annotation icons at the same time.
  • Once the desired annotations are selected, select the move icon in the top toolbar.
  • Drag anywhere on the screen with your finger to move the annotations the desired location.
  • Select the move icon in the top toolbar when finished.
  • Select the multi-select icon in the top toolbar to unselect the annotations.

Steps for Web Client

  • Select the multi-select icon in the top toolbar.
  • Tap on each annotation icon to select them, or drag to select multiple annotation icons at the same time.
  • Once the desired annotations are selected, select the move icon in the top toolbar.
  • Drag anywhere on the screen with your mouse to move the annotations the desired location.
  • Select the Done icon on the right side of the screen when finished.

7.5 Undo

Let's face it. We all make mistakes! The undo button allows users to revert the last several actions back in order to recover from virtually any activity performed within a document.

Any action performed within a document can be undone until the document is closed.

Danger

There is currently no undo function on the web client. This is a feature that will be added in a future enhancement.

There is no limitation on the number of actions that can be undone as long as the user remains within the document. Once the document is exited, the abilility to undo edits will be lost.

Warning

The undo action will not be able to recover activity once the user exits a document.

7.6 Redo

Let's face it. We all make mistakes. And sometimes the mistake was hitting the undo button! The redo button allows users to revert the last undo actions back in order to recover from virtually any activity that has been undone within a document.

Any undo action performed within a document can be redone until the document is closed.

Danger

There is currently no redo function on the web client. This is a feature that will be added in a future enhancement.

There is no limitation on the number of actions that can be redone as long as the user remains within the document. Once the document is exited, the abilility to redo edits will be lost.

Warning

The redo action will not be able to recover activity once the user exits a document.

7.7 Multi-Select

Sometimes it is beneficial to make the same edits to more than one annotation within a document. The multi-select toolbar button was designed to make this process easy.

Once a user has selected multiple annotations, a number of actions become available in the Data Panel. This may vary depending on which annotation icon types are selected.

Note

Tap on each annotation icon to select them, or drag to select multiple annotation icons at the same time.

Once the user is satisfied with the edits, they can select the mutli-select button again to turn it off. They will now notice that all of the edits that were performed have been saved with each annotation that was selected.

Warning

All properties can be modified when multi-select is enabled, including changing the annotation shapes.

7.8 Layers

Layers provide the ability to show and hide annotations within a document and serve as an imaginary stack of transparent slides on top of a document page.

The concept of adding layers is an important one, as each layer can be hidden or visible to the users. By hiding a layer, all of the annotations placed onto that layer are hidden from view as though they are not there. By unhiding the layer, they immediately reappear on the document.

Each new document comes with a Default layer, and all edits to the document are made on this layer until additional layers are created by the user.

  1. Close the Layer Panel screen
  2. The Default layer, which can be expanded and collapsed. Any new layers that are added will work in the same manner
  3. The show icon indicates whether this layer is visible or hidden on the document. When the icon is tapped, it will switch to have a line through it if it is hidden.
  4. Tapping the ... will bring up a window of additional options.
  5. The checkbox represents the current layer. The current layer is the layer on which any new annotations will be placed.
  6. When a layer is expanded, the annotations that have been placed onto the layer will appear, along with a variety of information about the annotation. Checking the box to the left of the annotation icon will result in the annotation being selected on the document page.
  7. Selecting the Add a New Layer button will expand the layer panel and present additional options to create new layers.
  8. Once a new layer is set up, click this button to save the layer to the document.
  9. The layer name is the one that will be displayed on the left side of this panel (see Level 1) and will be the name that appears in the layer selection window when changing the layer for an annotation.
  10. New layers can be added to all pages, the current page only, or any other page within a document.

7.8.1 Show and Hide Layers

Once layers are established, they can be set as visible or hidden. By hiding a layer, all annotations that have been placed upon that layer will be hidden on the document.

A good example of how this could be useful is when a project recurs over time. Users can create a layer for each visit, allowing them to hide or show the annotations from a previous visit.

7.8.2 More Options

Once created, layers can be locked, renamed and deleted.

  • Locking a layer prevents users from editing any annotation placed on the layer as well as placing any new annotations on the layer. When a layer is set to locked, it will display a lock icon next to the show and hide icon to indicate its status.
  • Renaming a layer allows users to change the name that appears in the Layer screen and when changing the layer for an annotation on a document.
  • Deleting a layer allows users to remove the layer from the document.

7.8.2.1 Locking Layers

If a user wants to view a layer within a document but not modify that layer in any way, including adding annotations or deleting existing annotations, they can lock the layer. Locking a layer prevents users from editing any annotation placed on the layer as well as placing any new annotations on the layer. When a layer is set to locked, it will display a lock icon next to the show and hide icon to indicate its status.

Warning

A layer will remain locked until the user unlocks the layer. Exiting from and returning to the document will not unlock the layer.

7.8.2.2 Renaming Layers

Renaming a layer allows users to change the name that appears in the Layer screen and when changing the layer for an annotation on a document.

Note

When renaming a layer that has been created on multiple pages, users will be provided the option to rename the layer on all pages within the document, the current page or any other page on the document simultaneously.

7.8.2.3 Deleting Layers

Deleting a layer allows users to remove the layer from the document.

Note

When a user deletes a layer, they are presented with the option of keeping any annotations that exist on that layer. Alternatively, they can delete all annotations and related information when the layer is deleted.

If you decide to keep the annotations, you will be asked to select the layer to move the annotations onto.

Warning

If a document page only has one layer, users will not be able to delete the layer. In addition, if the document only has one layer, users will not be able to select the option to Keep Annotations as there is no additional layer to move them on to.

7.8.3 Set Default Layer

The default layer is the layer on which any new annotations will be placed. When a new document is added to a project, it is automatically assigned a default layer named Default. All annotations added to the document will be placed on this layer until another layer is added and set as the default layer. Alternatively, annotations can be assigned to a different layer after they have been added to the document.

In order to set the default layer within the layer panel, simply check the box to the right of the layer. In order to view which layer is the current default layer, open the layer panel and look for the check box.

7.8.4 Select Annotations

Once inside the layer panel, users can expand and collapse the various layers to display the annotations that exist within each layer. By selecting the checkbox to the left of the annotation icon, the annotation will automatically be selected on the document page. This is useful when trying to locate an annotation on a page, or when looking to select multiple annotations on a page.

You may select multiple annotations from any number of layers on the page.

Note

If a layer is marked as hidden, any annotations selected from the hidden layer will not appear as they will also be hidden on the page.

7.9 Export a Document

Documents can be exported in a variety of formats. Users have the option of exporting each page or the entire document at once, and with both options users have the ability to include or exclude annotations.

The web client has two additional options - 1) Export to AutoCAD and 2) Download Media.

  1. Include Links - If this toggle switch is turned on, the Export page as PDF and Export document as PDF will include annotations that can be selected, resulting in Plannotate web client opening to the annotation icon on the selected report.
  2. Export Blank Page as PDF - For documents that are more than one page, the current page of the document can be exported to a new PDF document without any markups.
  3. Export Page as PDF - For documents that are more than one page, the current page of the document can be exported to a new PDF document with markups.
  4. Export Blank Document as PDF - For documents that are more than one page, the entire document can be exported to a new PDF document without any markups.
  5. Export Document as PDF - For documents that are more than one page, the entire document can be exported to a new PDF document with markups.
  6. Export Document as CSV - The entire document can be saved as a CSV flat file with all document contents stored into rows and columns.
  7. Export Document as Reporting CSV - A subset of the document can be saved as a CSV flat file similar to the Export Document as CSV, but with certain rows and columns pivoted.
  8. Export Photos - All annotations with photos will be exported to a CSV with the photo information and associated URL provided.
  9. AutoCAD Integration - All annotations we be exported to a JSON format in order to be imported into AutoCAD.
  10. Downloading Photos - All photos within the document or project will be exported and savid into a .zip file.

Note

With the release of 3.1.0, the export features will trigger a process to run on the server, and the user requesting the export will receive an email when the file is ready for download. This change was made in order to avoid waiting on very large projects to be exported.

7.9.8 AutoCAD Integration

Plannotate allows users to export annotations to a JSON format that can then be loaded into AutoCAD. The original CADapult tool that was created for allowing Plannotate annotation shapes to be imported from and exported to AutoCAD has been modified in order to support Plannotate version 3. In order to implement the updates, two files will need to be added to your computer. These files can be found within the CADapult.zip file located on Microsoft Teams in the Plannotate User Community shared space. Navigate to Documents > General > CADapult. You will also find the original CADapult - Plannotate AutoCAD plugin user guide at this location.

Note

The following steps assume that you have installed CADapult from the WJE network. If you have not, please see the CADapult - Plannotate AutoCAD plugin.pdf for steps on how to do this.

Installing New Files for Plannotate version 3

  1. Download the Cadapult.zip file to your computer from Teams.
  2. Unzip the files and place them in a new directory in the same location as where the CADapult plugin is currently installed.
    • Navigate to C:\Program Files\AutoCAD 2018\Plugins
    • Create a directory called p3 (or whatever you prefer)
    • Place the two files from the .zip file into the new p3 directory
  3. Since the files where not provided as part of an installation, they need to be unblocked by your computer in order to work.
    • Navigate to the two new files in C:\Program Files\AutoCAD 2018\Plugins\p3\
      • Cadapult.dll
      • Newtonsoft.Json.dll
    • Right click on the first file to find the properties (and then repeat these steps for the second file).
    • In the bottom right corner there is a checkbox for “Unblock”. Check the box if it is not already checked, and then select Apply.
    • Select OK and repeat for the second file.

Exporting AutoCAD content for Plannotate

You may continue to export content from AutoCAD to be imported into Plannotate version 2 or version 3 using the same set of instructions as was used in Plannotate version 2.

Importing Plannotate Annotations into AutoCAD

If you are importing Plannotate version 2 content into AutoCAD, you will also use the same set of instructions in the existing CADapult documentation.

If you are importing Plannotate version 3 content into AutoCAD, you will follow these modified instructions.

  1. Select the Export button within any Plannotate version 3 document and then select Export to AutoCAD.
  2. Copy the Document URL that is displayed to your clipboard.
  3. Launch AutoCAD and in the command line, type NETLOAD followed by the ENTER key.
  4. The Choose .NET Assembly dialog will appear.
    • Navigate to the C:\Program Files\AutoCAD 2018\Plugins\p3\ folder that was created above.
    • Select the new Cadapult.dll file that was placed there in the steps above.
    • Select the Open button.
  5. In the command line, type PLANNOTATE2 followed by the ENTER key.
  6. The CADapult JSON Importer dialog will appear.
    • Copy and Paste the Document URL from the Plannotate Document URL dialog in step 2 above into the line provided.
    • Leave the username and password fields empty. They are not needed for Plannotate version 3 content to be imported.
    • Select the Load Document button.
  7. Continue to Step 7 on page 5 of the CADapult - Plannotate AutoCAD plugin.pdf.

When the Manage Tags option is selected, a popup will appear displaying any tags that have been assigned to the page. These will show up in orange, for example "Concrete".

  • Point - Annotation shapes include circle, octagon, square, triangle, star, diamond, flag, camera, arrow and text.
  • Freehand - Annotation shapes include freehand line and freehand area.
  • Vertex - Annotation shapes include polyline arrow, polyline, line and line axial.

7.10.4 Hide Toolbar Annotation Names

Each annotation icon by default has a name displayed below the shape on the toolbar. These names can be disabled by selecting the Hide Toolbar Annotation Names option.

7.10.5 Set Scale

The Set Scale option allows you to define the length of a line on the page, which can then be used to calculate a variety of geometric values depending on the annotation icon that is selected. For example, the length of the scale line can be used to determine the area of a circle, whereby the radius of the circle can be determined based upon how big it is in comparison to the scale line.

When the scale for a page is set, a box is placed into the bottom right corner representing a legend that shows the length of the scale line and the value in feet that it represents.

  • Tap the ... button in the top menu of a project document.
  • Select the Set Scale option from the menu.
  • Draw a line on the document page to indicate the size of your scale. You can continue to tap and draw until you are satisfied with the length of the line..

    • Note

    The placement of the scale line does not matter, as once it is established, it will move to the bottom right corner of the document. Therefore, you can draw the line wherever you desire on the document page, focussing on the length of the line and not on the placement.

  • Enter a numeric value at the bottom of the screen to estalish how many feet the line represents.
  • The scale be be set independently for each page in the document or for all pages within the document.
    • If you want to set the scale for the current page only, uncheck the option for All Pages.
    • If you want to set the scale for all pages, check the option for All Pages.
  • Tap the Set Scale button.

Scale calculations apply to the following annotation types:

Circle annotations have a radius r and an area A that is calculated as follows:

A = Π r2

Octagon annotations have a side a and an area A that is calculated as follows:

A = 2 (1 + √2) a2

Square annotations have a side a and an area A that is calculated as follows:

A = a2

Triangle annotations have a base b, height hb and an area A that is calculated as follows:

A = hbb / 2

Diamond / Rhombus annotations have a diagonal p, a diagonal q and an area A that is calculated as follows:

A = p q / 2

Freehand Line annotations have a length measuring the perimeter of the line that is drawn.

Freehand Area annotations have a length l, a width w and an area A that is calculated as follows:

A = l w

Polyline-Arrow annotations have a length measuring the perimeter of all segments of the line that is drawn.

Polyline annotations have a length measuring the perimeter of all segments of the line that is drawn.

Drawn Ellipse annotations have an axis a, an axis b and an area A that is calculated as follows:

A = Π a b

Drawn Rectangle annotations have a length l, a width w and an area A that is calculated as follows:

A = l w

Line annotations have a length measuring the perimeter of the line that is drawn.

Line Axial annotations have a length measuring the perimeter of the line that is drawn.

7.10.6 View Only Mode

All projects have built in security (see Manage Users for more information). Though a user with Edit or Admin rights has the ability to mark up documents, there may be times where they want to view the document and annotation details without accidently modifying any of the content. This is the concept behind setting the View Only Mode checkbox.

When view only mode is enabled, the user cannot add annotations or edit any of the existing document markups. All information will be set to read-only mode.

Note

View Only Mode can be turned on and off as desired by each user, and does not have any impact on project security.

7.11 Copy a Document to Plannotate

In addition to adding a PDF file as a new document within the Plannotate interface, you can share a PDF document opened on your iPad's PDF viewer. This method is more involved than Adding a Document From Device.

  • Open the PDF document on your iPad.
  • Select the forward icon in the top right corner.
  • Select the Plannotate version 3 icon from the list of applications that appear.
  • You will be presented with a message indicating "Your file is copied to Plannotate 3. Please select any project and add to a folder.". Select OK.

Note

If a project is opened when performing this action, you will be required to exit to the Project Dashboard and then re-open the project before seeing the option to import the copied document.

  • Open the desired project where the document is to be added.
  • You will now see a blue button centered at the bottom of the screen that reads "Import copied document". Tap on this button to import the document.
  • Provide a name for the document or leave the file name as the document name.
  • Select Add.
  • You will be presented with a tree view of the existing folders and subfolders in the project. Tap the desired folder to add the new document into.

    • Warning

    It is advised that Plannotate is open and you are in the Project Dashboard prior to performing this action. If Plannotate is not open, preforming this action will open the application and have you log in as applicable.
