The Form Builder is the entry point to project forms, an area unique to each project, where users can create, modify and delete project forms. Forms go hand-in-hand with toolbars and enable users to add detailed annotation information to project documents.

Each new project comes with a Default form, which can be modified or deleted.

  1. Project Title –References the name of the currently selected project.
  2. Form Builder – The entry point to project forms.
  3. Support – Provides Plannotate support videos, documentation, and support ticket creation.
  4. Dashboard – Allows you to exit the project and return to the Project Management Dashboard.
  5. Form List – This is the space where the list of all project forms will be displayed.
  6. Search – Enables you to search the displayed list of forms based on several search criteria. See Project Management Search for more information.
  7. Sort – Enables you to sort the displayed list of forms based on several sort options. See Project Management Sort for more information.
  8. Export – Allows users to export project content to a variety of formats. See Export for more information.
  9. Manual Sync – Initiates a synchronization of data with the Plannotate cloud, which uploads all new data since the last sync, and downloads any new data from the Plannotate cloud. See Project Management Manual Sync for more information. This icon is available only on Plannotate for the iPad, as the web client version is communicating real time with the Plannotate cloud. Plannotate on web client has an Apple icon in the top right corner to allow users to install Plannotate on their iPad.
  10. Add – Provides the ability to add new forms.

5.1 Form List

The list of forms displays all project forms, sorted by default in alphabetical order.

When you first create a project, you will have a default form, and as new forms are added, they will appear in a list view.

Once new forms have been added, or existing forms have been modified, they will either show in a blue or orange color. The list will default in alphabetical order, and can be searched and sorted to change the order displayed.

Note

  • Forms will show in an orange color if they have been created or modified on your device and have not yet been synchronized to the Plannotate cloud. Note that this is only relevant to Plannotate on the iPad, as the web client version is communicating real time with the Plannotate cloud. This allows users to work “offline” and sync to the Plannotate cloud at any time when they are back online, manually or on a schedule. See Sync Settings for more information.
  • Forms will show in a blue color if they have been fully synchronized to the Plannotate cloud.

5.1.1 Form Options

There are two actions that can be performed on a form.

  • Single tap – Selects and opens the form into the Form Builder.
  • Long hold – Opens a menu of form options.

When a long hold is performed, a popup menu showing a variety of options will appear.

On the web client, a left click on your mouse button is used as a single tap, and a right click on your mouse button is used for a long hold.
  1. The title of the form that was selected.
  2. Cancels out of this screen. You can also select anywhere off the menu to close the screen.
  3. Rename - Allows you to rename the form.
  4. Manage Tags - Opens the Manage Tags screen.
  5. More Information - Displays form metadata / details.
  6. Hide - Enables the form to be hidden from use. See Hidden Content

5.1.1.1 Rename A Form

The user that initially creates a form will assign a name to the form. In the event this name needs to change, this option can be selected.

When the Rename option is selected, the current name of the form will appear, and by tapping on the data entry line, you will be able to modify or enter a new name.

  • Select Rename to change the name of the form.
  • Select the X in the top right corner to cancel out of this screen, leaving the form name unchanged.

5.1.1.2 Manage Tags

Tags are metadata assigned to a Plannotate object like a project, document, annotation, etc. By adding meaningful tags, users will be able to search for content more effectively.

When the Manage Tags option is selected, a popup will appear displaying any tags that have been assigned to the form. These will show up in orange, for example "Concrete".

  • Enter any number of tags on the data entry line, separated by a comma, and hit the enter key on your iPad keyboard to record them.
  • Select the X next to any tags in orange to remove them.
  • Select Save to save all the changes that were made.
  • Select the X in the top right corner to cancel out of this screen, leaving the tags unchanged.

5.1.1.3 Form Information

The More Information screen shows metadata information associated with the form and its contents.

  • The top section shows the form name, created data, last modification date and the tags associated with the form.
  • The main body of the screen shows how many form fields exist within the form.
  • Select the X in the top right corner to cancel out of this screen.

5.2 Adding Project Forms

The Add Form section of Form Builder allows you to create new forms as well as make a copy of existing ones from within the project or from other projects which you have access. Any user with appropriate access to a project can create a new form or copy other forms.

After tapping the + at the bottom right corner of the screen, a popup screen will appear allowing you to select from a few options.

  1. The title of the menu.
  2. Cancel out of this screen. You can also select anywhere off the menu to close the menu.
  3. New Form - Add a new form to the project.
  4. Duplicate Existing Form - Add a new form by making a duplicate of an existing form within the same project.
  5. Import Form From Other Projects - Add a new form by making a duplicate of an existing form from another project which you have access.

Note

If a form is created with the same name as an existing form, the name will be appended with a number, incremented by 1 each time this happens. For example, if a form named "Crack" exists and is created for a 2nd time, it will receive the name "Crack (1)". If this were to be added again, it would be given a name "Crack (2)" and so on.

5.2.1 Adding a New Form

After creating a project, you may want to add additional forms to capture annotation content. There are several methods to do this, the first of which is to create a brand new form.

  • Provide a name for the new form.
  • Select Add to add the form.
  • Select the X in the top right corner to cancel out of this screen.

5.2.2 Duplicate an Existing Form Within the Same Project

After creating a project, you may want to add additional forms to capture annotation content. There are several methods to do this, the second of which is to duplicate an existing form within the same project.

Note

In order to select a form to duplicate, check the box to the right of the desired form. You may only select one form at a time. Therefore, if you wish to add a copy of several forms, you will repeat these steps.

  • Select the form you wish to duplicate from the list of forms within the project.
  • Select the Create Copy button.
  • Provide a name for the new form.
  • Select Add to create the form.
  • Select the X in the top right corner to cancel out of this screen.

5.2.3 Import an Existing Form From a Different Project

After creating a project, you may want to create additional forms to capture annotation content. There are several methods to do this, the third of which is to import an existing form from a different project project.

Note

In order to select a form to import, check the box(es) to the right of the desired form. You may select multiple forms at once.

Warning

In order to import forms from other projects, it is important that you first perform a sync to ensure that the projects and forms have been downloaded to your device.

  • Select the forms you wish to import from the list of forms within the projects which you have access.
  • Select the Import button.
  • Select the X in the top right corner to cancel out of this screen.

5.3 Form Designer

Once a form is selected, the form designer screen appears, allowing you to create, modify and delete form fields.

The form designer is divided into three primary areas: the available field types, the body of the form being created, and the properties of each form field once selected.

  1. The name of the form that was selected.
  2. The full list of form fields that are available to add to the form.
  3. Support – Provides Plannotate support videos, documentation, and support ticket creation.
  4. Exit - Exists back to the main Form Builder screen.
  5. The main body of the form, where you will add, modify and organize your form fields.
  6. The properties area of the form, where you will set up and modify each form field property.
  7. Build – The build screen is the main screen of the Form Designer.
  8. Preview – Preview allows you to see what the form will look like as well as interact with the form to test the functionality.
  9. Publish – Forms are not saved until the Publish button is selected.

5.3.1 Adding Form Fields to a Form

Form fields can be added to the main body of the form by tapping on the form field on the left.

  • Locate the field to add to the form.
  • Tap on the field.

Note

If a form field is created with the same name as an existing field, the name will be appended with a number, incremented by 1 each time this happens. For example, if a field named "Length" exists and is created for a 2nd time, it will receive the name "Length (1)". If this were to be added again, it would be given a name "Length (2)" and so on.

5.3.2 Reorder Form Fields

Form fields are added in the order they are tapped, and can be easily reordered through drag-and drop.

  • Hold your finger on the three lines to the right of any field.
  • Drag the field to the desired location on the form.
  • Release your finger.
On the web client, a left click on your mouse button is used as a single tap, and a long hold on your left mouse button is used for a long hold.

5.4 Form Fields

Form fields comprise the most important part of forms. There are a variety of form field types, and each can be customized through the form field properties.

Single Line Text Used for single-line input and accept any form of text.
Text Area Used for multi-line input and accept any form of text.
Text Label Used as a header or label within the form.
Number Used for numeric value input and accepts any numeric value.
Divider A separator in order to divide form fields.
Checkbox Represents a single, predefined checkbox that a user can select.
Dropdown Provides a pre-defined list of options that a user can select.
Single Choice Represents a group of pre-defined radio buttons that a user can select.
Multiple Choice Represents a group of pre-defined checkboxes buttons that a user can select.
Date Picker Presents a date field as a calendar-style date picker UI that you choose a date from.
Formula Displays the result of a calculation that has been defined by the form creator.
Address Mutli-value text boxes with default address input fields.

5.4.1 Form Field Properties

There are a variety of form field properties shared by various form fields throughout the form builder. A complete list of the properties can be found below, along with a reference to the form fields that use each of them.

Property Description Applies To
Label Text The value that will be displayed above form field on a form and can be thought of as the name of the field. All except Divider
Label Align Determines the position of the field label to the field value. Choices are Top align or Left align. All except Divider
Excel Column Name Used in previous versions of Plannotate for column headers in CSV outputs. This feature has been depricated with Plannotate 3.x and is no longer applicable. All except Divider
Placeholder Text Descriptive text displayed inside an input field until the field is filled. It disappears when you start typing in the field. This is an optional field when available. Single Line Text, Text Area, Number, Date Picker
Default Value The value that will be used for an input field unless it is modified. Single Line Text, Text Area, Number, Dropdown, Single Choice, Date Picker. This is an optional field when available.
Use Conditions Determines if an action should take place on the field based upon the value set in another field within the same form. See Use Conditions. All except Text Label, Address and Divider.
Minimum Value Determines the lowest value accepted for a numeric field. Number

Options Determines the choices available to select, along with a variety of properties and actions that can be applied if the choice is selected. See Set Options. Dropdown, Single Choice, Multiple Choice
Default Date Allows today's date to be used in a date field. Date Picker
Calculation Allows a calculation to be defined, with elements including numbers, simple operations and other form field values. See Field Calculations Formula

There are several ways to create a calculation value, including but not limited to:

  • Tapping on simple operations including addition, subtraction, multiplication, division and parenthesis to group information within the formula
  • Adding the value from other form fields within the form. This can be done for any form that has either a numeric input (such as number), or a field that allows for a number to be assigned to a choice (such as the options within dropdown, single choice and multiple choice fields).

    • You can create any number of calculations using the key pad directly under the Calculation entry box.

    5.4.14 Form Field Conditions

    Form fields conditions enable limited if, then, else functionality to most form field types. When enabled, a form field can be hidden or displayed based upon the value entered in another field.

    Conditions can be enabled for all form field types with the exception of Text Label, Address and Divider. In order to enable the use of conditions, toggle the Use Conditions field to on for the applicable field.

    There are 4 values that must be set in order for the condition to work.

    Field Determining the Condition This is the field that will be used to determine how the condition will work. In an If, Then, Else statement, this is the first part of the If (If Field Determining the Condition).
    Condition

    This is the operation that will be used along with the Field Determining the Condition. In an If, Then, Else statement, this is the second part of the If (If Field Determining the Condition is equal to). Available choices are:

    • is Equal To
    • is Not Equal To
    • is Empty
    • is Filled
    Select a Value

    This is the operation that will be used along with the Condition. In an If, Then, Else statement, this is the third part of the If (If Field Determining the Condition is equal to value).

    Note

    If the is empty or is filled are selected for the Condition, then this value will not appear as it is not applicable.
    Select the Condition Action This value allows you to select if you want to show or hide this form field if the condition is met. In an If, Then, Else statement, this is the Then. The Else portion will always be the opposition of what is selected here. For example, if the choice is to show this field, then the else will be to hide this field.

    Note

    Multiple Choice fields cannot be used in determining the action to take in a condition as users can select multple values and the condition would not know how to address if multiple answers are selected.

    In the following example, we have a Single Choice field that asks the user their favorite fruit. The choices are Apple, Banana, Cherry, and Peach. If the user selects Apple, they are presented with a question asking which is their favorite color apple, Red or Green.

    • Set up a Single Choice field called Favorite Fruit and define the options to choose from as Apple, Banana, Cherry, and Peach.
    • Set up a Single Choice field called Favorite Color Apple and define the options to choose from as Red or Green.
      • Enable the radio button for Use conditions.
      • Select the field that will determine the condition. In this case that is the Favorite Fruit field.
      • Select the condition from the list of choices. In this case that is is Equal To.
      • Select the value to trigger the condition. In this case that is Apple.
      • Determine if you want to show or hide the Favorite Apple Color field if the value in the Favorite Fruit field is set to Apple. In this case, we will select Show.

    We can now preview the form and see how this works.

    5.4.15 Form Field Options

    Form field options enable users to:

    • Set the default value
    • Change the properities of an annotation icon
    • Assign a numeric value to the options, which can be used in a calculation within a Formula field.

    Note

    Form field options are available in Dropdown, Single Choice and Multiple Choice form field types.

    There are 4 columns where values can be assigned to options.

    Options This is where users can assign the values that will serve as choices. For example, in a question such as "What is your favorite color", the values could be something such as Red, Yellow, Green and Blue
    Default This allows users to determine which value(s) will automatically be selected when the form is accessed. Any user of the form can modify this value, however if this value is not changed, it will remain as the selected value.
    Calculated Values

    This column works in conjunction with the Use Calculated Values toggle. If the toggle is enabled, each Option added will have a value assigned to it. When the option is selected, the option value will be displayed, and the value will also be assigned but not displayed. This allows the form field to be used in a Formula field.

    In the example above where a user picks their favorite color, we could assign a value of 5 to Red, 10 to Yellow and 15 to Green. If the "What is your favorite color" is used in a Formula field where the formual is 1 x [What is your favorite color], the result will be 5 if Red is selected, 10 for Yellow and 15 for Green.
    Attributes

    Attributes are an optional value that can be selected. When selected, the user can determine if this option value will result in the annotation icon, fill color, stroke color, opacity and / or line width will be modified when selected.

    In the example above where a user picks their favorite color, a user could decide that if Red is selected, the annotation icon will change to a cirlce with a red fill color, black stroke color, opacity of 0.5 and line width of 5.0.

    When a user creates an annotation, selects this form to use with the annotation, and then selects Red for the "What is your favorite color" field, the annotation will be changed.

    Note

    Attributes cannot be assigned to option values in Mutliple Choice form fields as the attribtute would not know how to be applied if multiple answers are selected.

    5.4.16 Adding Calculations to Formula Form Fields

    The formula form field allows users to create calculations with simple arithmetic. The formulas can include numbers, operations such as add and subtract, as well as values from other form fields within the same form.

    1. Calculation Area - where the created formula will appear.
    2. Numeric buttons - adds the corresponding number to the formula.
    3. Operators - adds the corresponding operator to the formula.
    4. Back space - allows users to remove some or all of the calculation from within the Calculation Area.
    5. Add Fields button - allows users to add other form fields to the form to be used in the calculation. For number form fields, the numeric value entered will be used. If the Use Calculated Values is toggled on for Dropdown, Single Choice, and Multiple Choice fields, the values assigned to each option will be used.

    Note

    If a form field is created with the same name as an existing field, the name will be appended with a number, incremented by 1 each time this happens. For example, if a field named "Length" exists and is created for a 2nd time, it will receive the name "Length (1)". If this were to be added again, it would be given a name "Length (2)" and so on.

    5.4.17 Copy Form Fields

    Once a form field is created, it can be easily duplicated into a new field. The newly created form field can then be modified as desired.

    • Select the desired form field to duplicate.
    • Select the copy icon in the bottom right corner of the form field properties window.
    • Provide a name for the newly created form field.

    5.4.18 Delete Form Fields

    Once a form field is created, it can be deleted at any time to be removed from the form.

    • Select the desired form field to delete.
    • Select the delete icon in the bottom right corner of the form field properties window.
    • Confirm that the field should be deleted.

    Warning

    If any of the form fields are deleted, they will no longer be seen in any new annotation that uses the form. They will also be removed from any existing annotation that uses the applicable form, noting that they will not be deleted but rather hidden from view.

    5.5 Preview a Form

    At any time during the creation or modification of a form, users can preview what the form will look like when Published. This will allow the user to confirm that the form is working as expected, and will allow users to validate form values, selections, and calculations.

    5.6 Publish a Form

    Once the form is complete and ready for use, users can Publish the form. If the form is new, users will now be able to use it. If the form contains modifications to an existing form, the modifications will now be seen.

    Danger

    If a form is exited before publishing, all changes will be lost. This may be desired if the user is testing or trying new things. In the event a user selects Exit and has not published the form, a warning will be presented confirming this action is desired.
