The Project Dashboard is the entry point into Plannotate project content (documents, images, etc.). Users can tap on the name of any available project in the Project Management Dashboard in order to enter the Project screen.

  1. Project Title –References the name of the currently selected project.
  2. Documents – Provides a list of all content within the project.
  3. Forms – The forms screen is the entry way to the Form Builder and displays a list of all the forms available for use within the project.
  4. Toolbars – The Toolbars screen is the entry way to the Toolbar Builder and displays a list of all the toolbars available for use within the project.
  5. Hidden Content – Provides a list of all content which you have designated as hidden.
  6. Manage Users – Add, modify, and remove user access to the project.
  7. Sidebar Expand / Collapse – The Plannotate sidebar on the Project Dashboard can be expanded and collapsed to maximize real estate.
  8. Support – Provides Plannotate support videos, documentation, and support ticket creation.
  9. Dashboard – Allows you to exit the project and return to the Project Management Dashboard.
  10. Project Content – This is the space where the documents and folders will be displayed based what has been created within the project. Note that each project comes with a root folder denoted with the text "/". This default value can be changed using Rename Folder
  11. Search – Enables you to search the displayed list of content based on several search criteria. See Project Management Search for more information.
  12. Sort – Enables you to sort the displayed list of content based on several sort options. See Project Management Sort for more information.
  13. Export – Allows users to export project content to a variety of formats.
  14. Manual Sync – Initiates a synchronization of data with the Plannotate cloud, which uploads all new data since the last sync, and downloads any new data from the Plannotate cloud. See Project Management Manual Sync for more information. This icon is available only on Plannotate for the iPad, as the web client version is communicating real time with the Plannotate cloud. Plannotate on web client has an Apple icon in the top right corner to allow users to install Plannotate on their iPad.
  15. Add – Provides the ability to add new content.
  16. Reconcile Stub Photos – Provides the ability to reconcile stub photos that were taken on an iPad.

4.1 Project Documents

The Documents area of a project provides a list of all content created within the project. This includes folders, subfolders, and documents. Refer to the Add Content section for more information.

When you first open a project, you will not have any project content. Once you have added to your first document or folder, it will appear in a list view.

Once project content has been added, it will either show in a blue or orange color. The list will default in alphabetical order, and can be searched and sorted to change the order displayed.

Note

  • Project content will have a download icon if the project has been created on any other device and not yet downloaded to the device being used.
  • Project content will show in an orange color if it has been created or modified on your device and has not yet been synchronized to the Plannotate cloud. Note that this is only relevant to Plannotate on the iPad, as the web client version is communicating real time with the Plannotate cloud. This allows users to work “offline” and sync to the Plannotate cloud at any time when they are back online, manually or on a schedule. See Sync Settings for more information.
  • Project content will show in a blue color if it has been fully syncrhonized to the Plannotate cloud.

4.1.1 Folder Options

There are two actions that can be performed on folders and subfolders within a project.

  • Single tap – Selects and expands the folder.
  • Long hold – Opens a menu of options.
On the web client, a left click on your mouse button is used as a single tap, and a right click on your mouse button is used for a long hold.
  1. The name of the folder that was selected.
  2. Cancel out of this screen. You can also select anywhere off the menu to close the menu.
  3. Renames the folder.
  4. Opens the Manage Tags screen.
  5. Displays folder metadata / details.
  6. Enables the folder to be hidden from the Documents screens. See Hidden Content.

4.1.1.1 Rename A Folder

The user that initially creates a folder will assign a name to the folder. In the event this name needs to change, this option can be selected.

When the Rename option is selected, the current name of the folder will appear, and by tapping on the data entry line, you will be able to modify or enter a new name.

  • Select Rename to change the name of the folder.
  • Select the X in the top right corner to cancel out of this screen, leaving the folder name unchanged.

4.1.1.2 Manage Tags

Tags are metadata assigned to a Plannotate object like a project, document, annotation, etc. By adding meaningful tags, users will be able to search for content more effectively.

When the Manage Tags option is selected, a popup will appear displaying any tags that have been assigned to the folder. These will show up in orange, for example "Concrete".

  • Enter any number of tags on the data entry line, separated by a comma, and hit the enter key on your iPad keyboard to record them.
  • Select the X next to any tags in orange to remove them.
  • Select Save to save all the changes that were made.
  • Select the X in the top right corner to cancel out of this screen, leaving the tags unchanged.

4.1.1.3 Folder Information

The More Information screen shows metadata information associated with the folder and its contents.

  • The top section shows the folder name, created data, last modification date and the tags associated with the folder.
  • The main body of the screen shows how many occurrences of the various types of objects exist within the folder.
  • Select the X in the top right corner to cancel out of this screen.

  7. Makes an exact copy of the document with annotations, forms, images, etc.

4.1.2.4 Copy Document With Annotations

The copy document with annotations screen allows you to provide a name for the new document.

Once completed, an exact copy of the document will be made. This includes all annotations, links, forms and form data.

Note

If you wish to make a copy of the document only, not including the annotations, forms, images and links along with the document itself, you will want to select Copy Without Annotations from the menu.

4.1.2.5 Copy Document Without Annotations

The copy document without annotations screen allows you to provide a name for the new document.

Once completed, an exact copy of the document itself will be made. The new document will not include the annotations, forms, images, etc.

Note

If you wish to make an exact copy of the document including annotations, forms, images and links along with the document itself, you will want to select Copy With Annotations from the menu.

4.1.3 Moving Content

In order to move folders or documents to another location within the project tree, simply tap and hold on the three lines to the right of the object to move and drag with your finger. Once you've located the desired spot, let go of your finger to place the object in the desired location.

On the web client, long hold the left button on the mouse to initiate the move.

Note

  • Folders can be nested underneath another folder.
  • Documents can be placed under the Root or within any folder.

Note

When moving a folder or document to another folder, you must drop the source folder or document on top of the target folder. If you drop it somewhere within the folder, it will not move the object.

4.4 Hidden Content

Hidden content provides an area for users to place folders, documents, toolbars and forms that they do not want included in the project any longer. This content is not deleted from the Plannotate cloud.

When you first log into Plannotate, you will not have any content in the Hidden Content area. Once you designate content as hidden, it will appear in a list view.

Once content has been designated as hidden, it will be denoted with a special icon and will either show in a blue or orange color. The list will default in alphabetical order, and can be searched and sorted to change the order displayed.

  • Content will show in an orange color if it has been created or modified on your device and has not yet been synchronized to the Plannotate cloud. Note that this is only relevant to Plannotate on the iPad, as the web client version is communicating real time with the Plannotate cloud. This allows users to work “offline” and sync to the Plannotate cloud at any time when they are back online, manually or on a schedule. See Sync Settings for more information.
  • Content will show in a blue color if it has been fully synchronized to the Plannotate cloud.

Warning

Content is hidden globally within a project, and therefore any items hidden by one user will be hidden for all users of the project.

4.7 Adding Project Content

The Add Content section of Project Documents allows you to create new folders and documents. Any user with access to Plannotate can create a new project, add content, and add other users. By default, the user who created the project will have administrator rights to the project.

After tapping the + at the bottom right corner of the screen, a popup screen will appear allowing you to select the type of content to add.

  1. The title of the menu.
  2. Cancel out of this screen. You can also select anywhere off the menu to close the screen.
  3. New Folder - Add a new folder to the project.
  4. New Document from Device - Add a new PDF document from your iPad or cloud server to the project.
  5. New Document from Photo - Add a new PDF document from your iPad’s camera. Once a picture is taken and approved, it will be converted to a PDF document for annotating. This option is available only on Plannotate for the iPad.
  6. New Document from Camera Roll - Add a new PDF document from your iPad’s camera roll. Once a picture is selected, it will be converted to a PDF document for annotating. This option is available only on Plannotate for the iPad.
  7. New Field Sheet - Add a new document from the WJE Field Sheet template (portrait or landscape).

Note

If a document or folder is created with the same name as an existing document or folder, the name will be appended with a number, incremented by 1 each time this happens. For example, if a folder named "Project ABC" exists and is created for a 2nd time, it will receive the name "Project ABC (1)". If this were to be added again, it would be given a name "Project ABC (2)" and so on.

4.7.1 Adding Project Folders

After creating a project, you may want to create new folders to organize your documents and subfolders.

As folders are added, they can be placed at the root or nested undeneath an existing folder.

  • Provide a name for the new folder.
  • Select Add to create the folder.
  • Select the X in the top right corner to cancel out of this screen.
  • You will be presented with a tree view of the existing folders and subfolders in the project. Tap the desired folder to place the new folder into.

4.7.2 Adding Documents From a Device

Documents make up the main content within a Plannotate project. Any existing PDF document can be imported into the project.

When adding a new document from your iPad, you can locate any PDF document on your iPad’s storage or any other cloud location that you have connected to your iPad, such as iCloud, Google Drive, Dropbox, etc.

  • Navigate to the desired document.
  • Tap on the PDF document image.
  • Provide a name for the selected document. By default, the name will be the same name as how it is represented on your device or cloud storage location.
  • Select Add to import the document.
  • You will be presented with a tree view of the existing folders and subfolders in the project. Tap the desired folder to nest the new document into.

Note

  • The locations available for selecting a document depend on each iPad. iPad users can add any number of cloud drives such as Dropbox, OneDrive, Google Drive, etc. Any 3rd party cloud drives that are added to the iPad will be made available as a location when adding a new document.
  • You may select any PDF, JPG or PNG file to use as a new document

Warning

Once a document is added to the project, it will no longer be connected to the original document that was located on your device or cloud drive. That means that any updates to the document in Plannotate will not be reflected in your cloud drive, and any updates to the document in your cloud drive will not have any impact on the document in Plannotate.

4.7.3 Adding Documents From a Photo

In addition to adding a PDF file as a new document, you can use your iPad’s camera to take a picture and import that as a new document. Images from your camera will be converted to a PDF file when imported as a new document.

Note

This option is only available on Plannotate for iPad.

  • Take a picture using your iPad’s camera.
  • Once taken, you can select Retake or Use Photo.
  • Once you have selected Use Photo, provide a name for the new photo. By default, the name will be the name assigned by your device, and you can select and change it.
  • Select Add to convert the image to a PDF and import the new document.
  • You will be presented with a tree view of the existing folders and subfolders in the project. Tap the desired folder to add the new document into.

4.7.4 Adding Documents From Camera Roll

Similar to adding a document from your iPad’s camera, you can use your iPad’s camera roll to select a picture and import it as a new document. Images from your camera roll will be converted to a PDF file when imported as a new document.

Note

This option is only available on Plannotate for iPad.

  • Locate the image you would like to add.
  • Once you have selected, provide a name for the new document. By default, the name will be the name assigned by your device, and you can select and change it.
  • Select Add to convert the image to a PDF and import the new document.
  • You will be presented with a tree view of the existing folders and subfolders in the project. Tap the desired folder to add the new document into.

4.7.5 Adding a New WJE Field Sheet

Plannotate also makes available a portrait and landscape copy of the standard WJE Field Sheet template for you to add to any project.

  • Enter a name for the new field sheet document.
  • Specify how many pages you would like the blank field sheet template to contain.
  • Select whether you would like the Portrait or Landscape version of the template.
  • Select Add to import the new document.
  • You will be presented with a tree view of the existing folders and subfolders in the project. Tap the desired folder to nest the new document into.

Warning

Though there is a placeholder in the number of pages to add, you must enter in a number or you will not be able to add the document.

4.8 Export

Plannotate projects and project content can be exported to a variety of formats, and this can be done directly from the project dashboard or within any project document. When exporting from the project dashboard, users have the option to export the project contents to a CSV file (in two ways) as well as export photos to CSV. On the web client, users have an additional option to download all of the project photos.

  1. Export Project as CSV will generate a CSV file containing metadata information from the project, including document information, pages, annotations, etc. This output will match the CSV export option from previous versions of Plannotate.
  2. Export Project as Reporting CSV will generate a CSV file containing metadata information from the project, including document information, pages, annotations, etc. pivoted so that the project contents can be easily accessed using Pivot Tables in Excel. A video with instructions as well as a reporting template can be found on the Plannotate User Community within Microsoft Teams, or on Sharepoint HERE.
  3. Export Photo will generate a CSV file containing a listing of all photos within the project as well as the URL to access the photo within the Plannotate Viewer.
  4. Download Media allows Plannotate users to download all project photos to a local drive for access outside of Plannotate. This option is available only on Plannotate for web.
